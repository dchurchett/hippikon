Note: All downloads link to the sourceforget.net project download page.

Hippikon 4.0

Hippikon 4.0 brings the framework up to JDK 1.6 compatibility, taking advantage of generics and annotations. The major change is making ProtectedResource an Annotation rather than a class that has to be extended. We trade off some type-safety for ease of use; any java object can not be a protected resource simply by using the annotation.
  • hippikon-4.0.tar.gz - for unix
  • hippikon-4.0.zip - other platforms

Hippikon 3.3

Hippikon 3.3 provides major new functionality in the PermsNavigator GUI tool, having added the ability to create and edit XML Policy Store files as well as simply viewing them. No core framwork changes were made in order to change the GUI tool but extensive work was done on the tool itself, which is all written in Swing.

  • hippikon-3.3.tar.gz - for unix
  • hippikon-3.3.zip - other platforms
To run the PermsNavigator and take a tour of the new features, downlad the latest release, cd into the directory where you extracted hippikon and type:

ant run-gui
At this time no startup scripts have been included in the distribution.

Hippikon 3.0

The first version ported from the original code base that has been in production for over 3 years. Hippikon was Written for JDK 1.2.2 and has been tested on JDK 1.4. Hippikon 3.0 does not work or compile with JDK 1.5 and will not be moved forward with future versions i.e., JDK 1.2.2 will no longer be supported.
  • hippikon-3.0.tar.gz - for unix
  • hippikon-3.0.zip - other platforms
Both achive files extract to the following directory structure:

hippikon-3.0
  |
  + javadoc               (generated javadocs)
  + lib                   (compiled hippikon.jar plus dependencies)
  + license.txt           (the LGPL License in full)
  + README.txt            (general notes)
  + build.xml             (the main build file for the system)
  + Hippikon_Manual.pdf   (a PDF copy of the online developer manual)
  + src                   (the source code)
    |
    + java-src            (contains the hippikon source code)
    + test-src            (contains the hippikon test code)
       |
       + config           (test XML policy store and resource.policies)
To install, extract to the file system, copy lib/*.jar to your application classpath resolving any possible conflicts with other versions of junit, log4j or ant.

To launch the Swing GUI permissions viewer, ensure hippikon.jar and log4j.jar are in your classpath or pass to the JVM as the -classpath argument: 

java com.hippikon.security.PermsNavigator
You will then be able to navigate to one of the example policy store XML files stored under src/test-src/config.

Hippikon 3.0 Change Log

Very few changes were made, those being to repackage and remove some proprietary references. Other changes include:

  • The logging system was updated to use log4j and a log4j.conf is provided to get the component working out-of-the-box
  • The unit tests now use the latest version of JUnit to avoid all the pesky javac warnings due to the assert keyword having been introduced in the JDK 1.4
  • XML Parsing is delegated to the APIs bundled in the JDK
  • the Ant build script is updated to create the tar and zip bundles and to launch the Swing GUI for viewing XML policy store files
  • Added the LGPL license header to all files
  • Added getDefinedPrinicipals() to the PolicyStore interface to support the PermsNavigator

External Dependencies

The following APIs are bundled with Hippikon in the lib directory:
  • Junit 3.8.1
  • log4j-1.2.9
To build and run Hippikon you will need to have installed JDK 1.4 or 1.5 (as of Hippikon 3.1 or greater) and have the JAVA_HOME environment variable set to the installation directory. You will also need to have installed ANT 1.6.2 or greater (the build file was written for ANT 1.6.2) and have the ANT_HOME environment variable set to the ANT installation directory. You will also need to have $ANT_HOME/bin in your path. Ant should pickup the right version of java from the JAVA_HOME variable but if you have another version of Java in your path you may have problems so check here first.

You will need to have junit.jar (located in the lib directory of the hippikon release) in your $ANT_HOME/lib directory in order to use the build script from the command line. If you are using an IDE see the IDE documentation for setting up sources and classpaths in order to get the build script working.

See the README.txt file bundled with the download package for general release details.

Obtaining The Source

Anyone may obtain the source code by downloading one of the official releases, or from the CVS repository. Instructions on using CVS for Hippikon can be found on the sourceforge CVS project page.

To obtain the java source files, checkout the module named 'authorization' off the TRUNK for the latest code (may not be stable) or a known tag, which can be found by browing the CVS repository using sourceforge's browser tool (located on the same CVS project page).
